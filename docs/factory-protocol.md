# `.factory/` Protocol Specification

The `.factory/` directory is the shared contract between all Attacca modes (Stand, Podium, Arco). Every project managed by Attacca has a `.factory/` directory at its root.

## Directory Structure

```
.factory/
├── CLAUDE.md              # Agent instructions (auto-generated by Stand)
├── config.yaml            # Project metadata (name, tier, phase, stack)
├── context.md             # Human-readable project context
├── status.json            # Machine-readable state (scanner reads this)
├── spec.md                # Project specification
├── intent-contract.md     # Intent contract (Tier 3-4)
├── scenarios.md           # Behavioral test scenarios
├── queue.json             # Work items (Podium writes, Stand reads)
├── sync-status.json       # Spec↔Tests↔Code triangle state
├── progress/              # Session logs (Stand writes)
│   └── session-YYYY-MM-DD-NNN.json
├── decisions/             # Architectural decision records
│   └── dec-NNN.md
└── artifacts/             # Generated outputs
```

## Types

All types are defined in `packages/contracts/src/factory.ts` and exported from `@t3tools/contracts`.

## Ownership

| File | Writer | Reader |
|------|--------|--------|
| config.yaml | Forge, Stand, Podium | All |
| CLAUDE.md | Stand (auto-gen) | Agent |
| context.md | Developer, Forge | All |
| status.json | Stand, Podium | Podium, Arco |
| spec.md | Forge, Developer | Podium, Stand, Agent |
| queue.json | Podium | Stand |
| sync-status.json | Podium, Stand | Podium, Stand |
| progress/* | Stand | Podium, Arco |
| decisions/* | Developer, Agent | All |
